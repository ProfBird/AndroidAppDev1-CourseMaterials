USCTide


The C# ASP.NET web service provides tidal water level prediction for U.S. coastal states and in Local Standard Time. Tide prediction is available at 300 tide stations along U.S. east and west coasts, Alaska, and Hawaii, from 1970 to 2037. Tide prediction from USCTide is NOT FOR NAVIGATION. Tide knowledge is from NOAA and XTide. USCTide was funded by NASA.

The following operations are supported. For a formal definition, please review the Service Description.

  • GetHarmonicTide
    The webmethod will return a single tidal water level. Time at input/output here is Local Standard Time.

  • GetHarmonicTides
    The webmethod will return an array of tidal water levels. Each element of the array is a TimeTide object containing (year, month, day, hour, minute, tideHeight). The TimeTide object together with the USCTide web service is accessible to web applications. The integer stationOrder inside (soap:Body) ... (/soap:Body) must be one record of StationID column in the table StationInventory in TideStations.mdb database. Time at input/output is Local Standard Time, instead of Coordinated Universal Time. Time Step is time interval in tide predictions, in minutes. The first character or the first two characters of the string timeStepInMintue inside (soap:Body) ... (/soap:Body) should be numeric, e.g. 6 minutes or 60 minutes. For consecutive two years of tide prediction, Time Step = 60 minutes. For one year of tide prediction, Time Step >= 30 minutes. For two months of tide prediction, Time Step >= 5 minutes. For one day of tide prediction, Time Step >= 1 minutes. If you want Time Step < 1 minutes or an example source code of a web application consuming the USCTide web service, you may ask Dr. Michael E. Hodgson at 803.777.8976.
